What it does

From wall-of-text answer to explorable image

Instead of a long text block, ChatImage gives you an image you can click around to learn only what you care about.

🎯

Clickable hotspots

Transparent, accurate regions overlaid on the generated image. Click any region to open its detail panel without losing your place.

🧩

Per-region detail & threads

Each hotspot keeps its own title, summary, and a follow-up thread — ask about a single region in context, not the whole answer.

👁️

Vision-aligned positioning

In real-API mode, LocateAnything / MiMo-vision / local OCR localize the actual visual regions so hotspots land on the right content — not a hard-coded grid.

🔌

Provider-agnostic

Run fully in mock mode with no keys, or proxy real text, image, and vision providers through the backend so keys stay server-side.

💾

Local-first persistence

Generated ChatImages, hotspots, calibration data, and follow-up threads are stored in a local SQLite database. Your work survives restarts.

📎

File context

Attach code, Markdown, CSV, JSON, or log files and include their contents as prompt context for more grounded answers.

Under the hood

How a ChatImage is made

Ten steps from your question to an explorable image.

You ask a question

Plain natural language in the composer — e.g. "compare REST and GraphQL".

Get a raw LLM answer

The text provider returns a long-form answer (or a deterministic mock answer).

Normalize into a visual spec

The answer becomes structured modules + auxiliaryModules, each a clickable region.

Plan a layout

A LayoutSpec assigns each module a region and normalized bounds.

Generate the image

An image prompt is derived from the structured content and the image provider draws the visual.

Align to real regions

Vision grounding (LocateAnything → SAM3 refine) localizes where each module actually rendered.

Overlay hotspots

Transparent, clickable hotspots are placed on the image at the grounded coordinates.

Click to explore

Clicking a hotspot opens its detail panel and follow-up thread.

Calibrate if needed

Toggle calibration mode to inspect and correct hotspot bounds.

Persist locally

The result and its threads are saved to SQLite for later review.

Get started

Run it locally in 3 commands

No API keys required — mock mode generates the demos above out of the box.

$ git clone https://github.com/wencanjiang/ChatImage.git $ cd ChatImage $ npm install $ npm start # then open in your browser: # http://127.0.0.1:5178?provider=mock # to use real providers, add keys: $ cp .env.example .env.local # edit .env.local $ npm start
  1. Install Node.js 18+ and npm.
  2. Clone and npm install.
  3. npm start launches the local server on port 5178.
  4. Open with ?provider=mock for instant, key-free demos.
  5. Want real models? Copy .env.example to .env.local, fill in your keys, and restart.
  6. Build the frontend with npm run build → outputs dist/.
