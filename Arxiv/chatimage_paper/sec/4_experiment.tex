We evaluate \method{} along two axes. The first axis measures end-to-end
completion: whether the system produces a complete ChatImage, consisting of a
generated picture and its hotspot layer, for ordinary prompts. The second axis
measures visual alignment evidence: whether a clickable region is supported by
a real grounding source and usable mask-backed preview data. All numbers below
come from a real-provider evaluation over the benchmark, with upstream API
quota interruptions treated as external run-limit events rather than pipeline
failures.

We assemble a benchmark of 30 questions designed to exercise different visual
modes and content types, summarized in Table~\ref{tab:benchmark_split}. The
current public showcase is a curated subset of the same workflow after strict
visual-alignment screening: a hand-drawn West Lake tour map and daily-life
illustrated scenes such as healthy breakfast options, boutique coffee shop,
sunny reading nook, independent record-store corner, and indoor plant-care
corner. Map questions stress geographic placement and route-like scenic areas;
scene questions stress object localization in less rigidly structured images;
infographic questions stress compact analytical panels and decision-support
layouts. Each benchmark question is run through the full \method{} pipeline
with real text, image, and vision providers. Only outputs that pass the
stricter public-demo gate are published on the demo page.

Because the grounding step is provider-agnostic
(Section~\ref{sec:method_alignment}), different regions can be aligned by
different signals: a dedicated localizer, a VLM grounding fallback, OCR, a
SAM-refined planned box, or a planned fallback. We therefore report not only
the final completion rates, but also where hotspot alignments actually came
from. Representative qualitative results are shown in
Figure~\ref{fig:qualitative}.

\subsection{Pipeline Results}
\label{sec:experiments_pipeline}

\input{table/main_results}

Figure~\ref{fig:experiment_summary} gives a compact visual summary of hotspot
quality, alignment-source mix, and the SAM mask-completeness diagnostic.

All 30 benchmark questions completed the full pipeline in valid runs, yielding
a generated-image completion rate of \textbf{$100.0\%$}. We treat this as a
sanity check for valid API runs rather than as the main quality result. The
more informative measurement is hotspot reliability: Table~\ref{tab:main_results}
uses a shared 24-hotspot checked subset to report both accepted and rejected
cases under the strict visual-alignment gate. The gate accepted \textbf{17 of
24 checked hotspots} ($70.8\%$) and rejected 7 ($29.2\%$). This gate is
intentionally conservative: each accepted hotspot needs a primary visual
grounding signal and a valid organic mask preview, so weak or missing visual
evidence is rejected rather than promoted. Within the same 24 checked hotspots,
\textbf{13} pass the SAM segmentation completeness check ($54.2\%$), while 11
remain incomplete ($45.8\%$), meaning the mask has obvious holes or empty
cavities.

\subsection{Alignment Source Analysis}
\label{sec:experiments_sources}

\input{table/ablation_results}

Table~\ref{tab:ablation_results} reports the grounding source distribution for
90 logged hotspot alignments from the diagnostic source sample. Most logged
hotspots are supplied by primary visual grounding: MiMo-Vision accounts for 50
hotspots ($55.6\%$), while LocateAnything contributes another 11 hotspots when
combining layout-guided and crop-based calls ($12.2\%$). SAM3-refined planned
regions account for 16 hotspots ($17.8\%$), and planned fallback is used for
11 hotspots ($12.2\%$). The source distribution shows that the system can
distinguish visually grounded regions from fallback regions: it records which
signal placed each hotspot, keeps high-confidence visual groundings when
available, refines plausible boxes with masks, and falls back to planned bounds
only when stronger evidence is unavailable.
